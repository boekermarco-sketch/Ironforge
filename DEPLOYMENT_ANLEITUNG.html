1
GitHub-Repository prüfen
~5 Min

Render braucht zwei Dateien im Repo-Root: requirements.txt und eine Procfile (startet die App).

Schritt 1a – requirements.txt prüfen
Deine requirements.txt existiert bereits. Öffne sie kurz und stelle sicher, dass diese Zeilen drin sind:

fastapi uvicorn[standard] sqlalchemy python-multipart pdfplumber python-docx requests requests-oauthlib garminconnect

Schritt 1b – Procfile erstellen
Render braucht eine Datei namens Procfile (kein Dateiendung!), die sagt wie die App startet. Diese Datei existiert in deinem Projekt noch nicht – du musst sie anlegen.

Öffne Notepad, tippe genau das:

web: uvicorn app.main:app --host 0.0.0.0 --port $PORT

Speichere die Datei als Procfile (kein .txt!) im Projektordner C:\Users\boeke\Meine Ablage\ClaudeAI\Fitness\.

⚠
In Notepad auf „Speichern unter" → Dateiname: Procfile → Dateityp: Alle Dateien (*.*) wählen → Speichern. Sonst wird es Procfile.txt und Render erkennt es nicht.

Schritt 1c – Procfile auf GitHub pushen
Gehe auf deiner Import-Seite auf „↑ Zu GitHub pushen". Fertig.

4
Umgebungsvariablen eintragen
~5 Min

Das sind deine Passwörter und API-Keys aus der .env-Datei. Du trägst sie direkt in Render ein – so bleiben sie sicher und kommen nie auf GitHub.

Scrolle auf der Seite nach unten bis zum Abschnitt „Environment Variables" und trage diese Werte ein:

Key (linkes Feld) Value (rechtes Feld)
GARMIN_EMAIL deine Garmin E-Mail-Adresse
GARMIN_PASSWORD dein Garmin Passwort
GITHUB_TOKEN dein Classic PAT (ghp_...)
SUPABASE_URL deine Supabase URL (falls vorhanden)
SUPABASE_ANON_KEY dein Supabase Key (falls vorhanden)
⚠
Withings-Credentials: Die .withings_credentials.json-Datei kann nicht als Umgebungsvariable eingetragen werden. Du musst sie nach dem ersten Deploy manuell hochladen (ähnlich wie Schritt 6 für die Datenbank). Alternativ: Withings-Abruf erstmal weglassen und nur Garmin nutzen.

6
Datenbank hochladen (einmalig)
~3 Min

Deine fitness.db enthält alle bisherigen Logs. Wenn sie im Git-Repo ist, wurde sie automatisch mitgenommen. Falls nicht, kannst du sie so hochladen:

ℹ
Option A (einfacher): Die fitness.db ist in deinem GitHub-Repo → sie wird automatisch deployed. Prüfe ob sie in .gitignore steht – wenn ja, entferne den Eintrag und pushe neu.
⚠
Wichtiger Hinweis: Render's Free Tier hat kein persistentes Dateisystem. Das bedeutet: Neue Daten die du online abrufst (Garmin, Withings) gehen verloren sobald der Service neu startet.

Lösung: Nutze die App primär lokal für Datenabruf und Import. Online-Zugriff eignet sich zum Ablesen der Dashboard-Ansicht. Für vollständige Persistenz wäre ein Upgrade auf Render Starter (7$/Monat) nötig.

Häufige Probleme & Lösungen
🔴 Deploy schlägt fehl mit „No module named X"
Ein Python-Paket fehlt in requirements.txt. Schaue welches Modul gemeint ist und füge es in requirements.txt ein, dann neu pushen.
🔴 „uvicorn: command not found"
Stelle sicher dass uvicorn[standard] in requirements.txt steht (mit eckigen Klammern).
🟡 Die Seite lädt 30 Sekunden beim ersten Aufruf
Normal! Der Free-Tier schläft nach 15 Min Inaktivität ein. Der erste Aufruf weckt ihn auf. Danach reagiert er sofort.
🟡 Garmin-Abruf schlägt online fehl
Garmin kann bei unbekannten IP-Adressen (Render-Server) eine Zwei-Faktor-Bestätigung verlangen oder kurz sperren. Garmin-Abruf lokal machen und Datenbank über GitHub aktuell halten.
🔴 „sqlite3.OperationalError: unable to open database file"
Die Datenbank wurde nicht gefunden. Prüfe ob fitness.db im Repo-Root liegt und in .gitignore nicht eingetragen ist.
🟡 Ich möchte das Dashboard mit einem Passwort schützen
Render bietet unter „Settings" → „Basic Auth" eine einfache Passwort-Absicherung an (HTTP Basic Auth). Das reicht für persönlichen Gebrauch vollkommen aus.
🟡 Neue Daten online sind nach Neustart weg
Das ist das persistente Dateisystem-Problem des Free Tiers (erklärt in Schritt 6). Für die Ansicht/Ablesen ist das kein Problem – nur Datenabruf lokal machen.
🟢 Ich möchte eine eigene Domain (z.B. marco-fitness.de)
In Render unter „Settings" → „Custom Domains" eintragen. Du brauchst eine Domain (~10€/Jahr bei Namecheap/INWX) und richtest dort einen CNAME auf deine Render-URL ein. Render stellt dann automatisch HTTPS aus.
