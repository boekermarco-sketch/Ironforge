Ironforge Health Sync

iOS Shortcut Anleitung – Apple Health → Supabase (täglich automatisch)

Was dieser Shortcut macht

  • MFP Kalorien, Protein, Kohlenhydrate, Fett aus Apple Health lesen
  • Withings Körpergewicht, Körperfett lesen
  • Garmin Ruhepuls, Schritte lesen
  • Supabase Alle Werte direkt in die Cloud speichern
  • Automation Läuft täglich um 23:30 Uhr automatisch
Voraussetzung: In MyFitnessPal muss die Apple Health Verbindung aktiv sein.
Wo: MFP App → unten rechts Profil → Einstellungen (Zahnrad) → Apps & Geräte → Apple Health → alles EIN
1
Supabase Tabelle anlegen (einmalig, am PC)
1
Supabase öffnen und SQL ausführen

Gehe auf supabase.com → dein Projekt → SQL Editor (links in der Navigation).

Kopiere diesen Code, füge ihn ein, klicke Run:

CREATE TABLE IF NOT EXISTS apple_health_daily (
  date          DATE PRIMARY KEY,
  calories      FLOAT,
  protein_g     FLOAT,
  carbs_g       FLOAT,
  fat_g         FLOAT,
  body_mass_kg  FLOAT,
  body_fat_pct  FLOAT,
  steps         INTEGER,
  resting_hr    INTEGER,
  sleep_min     INTEGER,
  synced_at     TIMESTAMP DEFAULT now()
);

-- RLS: Anon darf einfügen und überschreiben
ALTER TABLE apple_health_daily ENABLE ROW LEVEL SECURITY;

CREATE POLICY "anon insert" ON apple_health_daily
  FOR INSERT TO anon WITH CHECK (true);

CREATE POLICY "anon update" ON apple_health_daily
  FOR UPDATE TO anon USING (true);
Fertig wenn: Du siehst "Success. No rows returned" — das ist korrekt, die Tabelle wurde angelegt.

Das ist die wichtigste Sache zu verstehen. Tippe auf den großen blauen "+" Button in der Mitte der leeren Fläche.

3
Die 9 Actions einrichten (nacheinander)
A1
Datum holen

Tippe auf + → Suche nach: Aktuelles Datum

Tippe auf "Aktuelles Datum" → wird hinzugefügt.

In der Action auf "Datum und Uhrzeit" tippen → "Datum" wählen

📅 Aktuelles Datum
Format Datum ◂
A2
Kalorien lesen (von MFP)

Tippe auf + → Suche: Gesundheitsdaten suchen

Tippe auf "Gesundheitsdaten suchen". Jetzt siehst du diese Action:

❤️ Gesundheitsdaten suchen
Typ Nahrungsenergie ◂
Zeitraum Letzte 3 Tage ◂
Sortieren Neueste zuerst ◂

So einrichten:

  • Typ: tippen → suche Nahrungsenergie → wählen
  • Zeitraum: tippen → "Letzte" → 3 → "Tage"
  • Sortieren: Neueste zuerst (so kommt heutiger Wert zuerst)
  • Limit: auf 1 setzen (nur letzten Eintrag)
A3
Kaloriewert aus dem Ergebnis extrahieren

Tippe auf + → Suche: Details der Gesundheitsdaten

Tippe auf "Details der Gesundheitsdaten abrufen".

🔍 Details der Gesundheitsdaten
Eigenschaft Wert ◂
Von Gesundheitsproben ◂

Eigenschaft: Wert  |  Von: Gesundheitsproben (die Variable aus A2)

Tipp: "Gesundheitsproben" ist die violette/blaue Variable die automatisch aus der vorherigen Action erscheint. Tippe auf das Feld und wähle sie aus.
A4–A8
Protein, Kohlenhydrate, Fett, Gewicht, Ruhepuls (gleich wie A2+A3)

Wiederhole jeweils die beiden Actions (Suchen + Details) mit diesen Typen:

Action Typ suchen nach Limit
A4 – Protein Protein 1
A5 – Kohlenhydrate Kohlenhydrate 1
A6 – Fett Fett (Gesamt) 1
A7 – Körpergewicht Körpermasse 1
A8 – Ruhepuls Ruhepuls 1
A9 – Schritte Schrittanzahl 1
Wichtig bei Schritte: Limit auf 1 setzen, Zeitraum "Heute". Schritte kommen manchmal als viele kleine Einträge — wir holen nur den letzten Tageswert.
A10
Wörterbuch (JSON-Paket) zusammenbauen

Tippe auf + → Suche: Wörterbuch

Tippe auf "Wörterbuch". Du siehst eine leere Action mit "Element hinzufügen".

📚 Wörterbuch
date Datum ▸
calories Wert ▸
protein_g Wert ▸
+ Element hinzufügen

Füge folgende Einträge hinzu — tippe jeweils "Element hinzufügen":

Key (links eintippen) Wert (rechts — Variable wählen)
dateVariable: Aktuelles Datum (aus A1)
caloriesVariable: Wert (Kalorien, aus A2+A3)
protein_gVariable: Wert (Protein)
carbs_gVariable: Wert (Kohlenhydrate)
fat_gVariable: Wert (Fett)
body_mass_kgVariable: Wert (Körpermasse)
resting_hrVariable: Wert (Ruhepuls)
stepsVariable: Wert (Schritte)
Wie wähle ich eine Variable? Tippe auf das Wertfeld (rechts) → es erscheinen unten die zuletzt verwendeten Variablen. Tippe auf die passende. Wenn du mehrere "Wert"-Variablen siehst: halte sie gedrückt um zu sehen welche Action sie kommt.
A11

Tippe auf + → Suche: Inhalt von URL abrufen

5
Shortcut testen
✓
Ersten Testlauf machen

Gehe in der Shortcuts App auf "Ironforge Health Sync" → tippe auf das ▶ Play-Symbol.

Der Shortcut läuft ca. 3–5 Sekunden. Wenn kein Fehler-Popup erscheint: Erfolg.

Prüfen ob Daten angekommen sind:

  • Supabase → Table Editor → apple_health_daily
  • Dort sollte ein Eintrag mit dem heutigen Datum stehen
Fehler "No permission": Die RLS-Policies aus Schritt 1 nochmal ausführen.
Fehler "404 Not Found": Die URL prüfen — kein Leerzeichen am Ende.
Leere Werte: MFP hat heute noch keine Daten geschrieben — morgen nach dem ersten MFP-Eintrag nochmal versuchen.
